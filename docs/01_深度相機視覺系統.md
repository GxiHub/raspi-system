# 深度相機視覺系統

**建立日期：** 2026-03-04　｜　**狀態：** 🟢 運作中

---

## 系統概覽

| 項目 | 內容 |
|------|------|
| 硬體 | Intel RealSense D435i 深度相機 |
| 部署位置 | Pi5（pi53 @ 192.168.1.111） |
| 外網網址 | https://vision.tfooddata.com |
| 服務名稱 | `arm-vision.service`（systemd，開機自啟） |
| 推理模型 | YOLOv8 nano（物件偵測 + 3D 座標） |

---

## 硬體規格

- **型號**：Intel RealSense D435i
- **序號**：140122071092
- **連接方式**：USB 3.2（Bus 002）
- **串流解析度**：848 × 480，30fps
- **深度範圍**：0.1m ～ 10m

---

## 系統架構

```
Pi5 開機
  │
  ▼
[udev] 自動釋放 usbhid 驅動（讓相機可被 pyrealsense2 控制）
  │
  ▼
[systemd] 啟動 arm-vision.service
  │
  ├─ 載入 YOLOv8 nano 模型（~3 秒）
  ├─ 連接 D435i：開啟 RGB + 深度雙串流
  └─ 主迴圈（背景 thread，持續運作）
        │
        ├─ 每幀：RGB + 深度對齊
        ├─ 每幀：YOLOv8 物件偵測
        ├─ 每幀：計算每個物件 3D 相機座標（X / Y / Z 公尺）
        └─ 結果存入記憶體，供網頁即時讀取
```

---

## 外網存取架構

```
使用者瀏覽器
  │  HTTPS（任何網路）
  ▼
Cloudflare（全球 CDN + TLS 終止）
  │  加密隧道
  ▼
Cloudflare Tunnel（cloudflared，不需開防火牆）
  │  http://localhost:5003
  ▼
Flask Web Server（arm_vision.py，pi53）
  │
  ├─ GET /          → HTML 頁面
  ├─ GET /stream    → MJPEG 即時影像（RGB + YOLO 標注）
  ├─ GET /depth     → MJPEG 深度彩色圖
  └─ GET /detections → JSON（所有偵測物件含 3D 座標）
```

## API 說明

### `GET /detections`

回傳目前畫面中所有偵測到的物件：

```json
{
  "count": 1,
  "fps": 3.1,
  "streaming": true,
  "detections": [
    {
      "class_name": "airpods",
      "conf": 0.57,
      "pixel": [782, 167],
      "camera_xyz": [0.285, -0.058, 0.489],
      "bbox": [758, 131, 804, 203]
    }
  ]
}
```

| 欄位 | 說明 |
|------|------|
| `class_name` | 偵測到的物件類別 |
| `conf` | 信心值（0~1） |
| `pixel` | 畫面中心像素座標 [x, y] |
| `camera_xyz` | 相機座標系 3D 位置（單位：公尺）|
| `bbox` | 邊界框 [x1, y1, x2, y2] |

## 關鍵設定檔

| 檔案 | 路徑 | 說明 |
|------|------|------|
| 主程式 | `~/projects/robot_arm/code/arm_vision.py` | Flask + RealSense + YOLO |
| systemd service | `/etc/systemd/system/arm-vision.service` | 開機自啟、自動重啟 |
| udev rule | `/etc/udev/rules.d/99-realsense-hid-unbind.rules` | 解決 USB 權限問題 |
| Cloudflare config | `/etc/cloudflared/config.yml` | Tunnel 路由設定 |

---

## 已解決的問題

| 問題 | 原因 | 解法 |
|------|------|------|
| `failed to set power state` | kernel `usbhid` 驅動佔用相機 HID 介面 | udev rule 自動 unbind |
| arm_vision 反覆崩潰 | `depth_frame` 過濾後型別變 `frame`，無 `get_distance()` | 加 `.as_depth_frame()` 轉型 |
| 兩進程搶相機 | nohup + systemd 同時跑 | 統一用 systemd 管理，停用 nohup |
